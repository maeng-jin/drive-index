# drive-index

오프라인 외장하드의 파일 목록을 미리 캐시해두고, 드라이브를 연결하지 않고도 빠르게 검색·열람하는 뷰어.

여러 개의 외장하드를 굴리다 보면 "이 파일이 어느 드라이브에 있더라?" 하는 순간이 생깁니다. 매번 드라이브를 다 꽂아보는 대신, 각 드라이브의 파일 목록을 한 번씩 텍스트로 떠두고 브라우저에서 통합 검색하세요. 로컬 단독으로도 쓸 수 있고, 가벼운 HTTP 서버로 띄워서 외부 PC·스마트폰에서도 같은 캐시를 조회할 수 있습니다.

## 구성

| 파일 | 역할 |
|------|------|
| `list_drive_files.py` | 드라이브를 재귀 스캔해서 파일 목록(.txt)을 만드는 Python 스크립트 |
| `file_tree_viewer.html` | 트리 뷰·검색·통계 UI. 로컬 드래그 앤 드롭과 서버 자동 로드 모두 지원 |
| `server.py` | 캐시 디렉토리를 HTTP로 서빙하는 가벼운 서버 (표준 라이브러리만 사용) |
| `capacity.example.json` | 드라이브 라벨·용량 정보를 정의하는 예시 JSON |

## 두 가지 사용 모드

### A. 로컬 모드 (단일 PC)

`file_tree_viewer.html`을 브라우저에서 바로 열고, `.txt` 파일들과 이미지·JSON을 드래그 앤 드롭하면 끝. 서버 없이도 모든 기능이 동작합니다.

### B. 서버 모드 (LAN에서 공유)

PC 한 대에서 `server.py`를 띄워 캐시 디렉토리를 노출하면, 같은 네트워크의 다른 PC·태블릿·폰에서 브라우저로 접속해 같은 카탈로그를 조회할 수 있습니다. 페이지를 열면 자동으로 캐시 목록을 불러옵니다.

## 빠른 시작

### 1. 드라이브 스캔

```bash
python list_drive_files.py E:\ -o a.txt --with-size
```

- 결과 파일에는 자동으로 타임스탬프가 붙어 `a_YYMMDD_HHMM.txt` 형태로 저장됩니다.
- `--with-size` — 파일 크기까지 기록 (뷰어의 용량 통계·게이지 표시에 필요)
- `--no-count` — 사전 카운팅 생략 (즉시 시작, 진행률 % 대신 경과 기준)
- `--include-hidden` — 숨김 파일 포함
- `--flat` — 그룹 포맷(기본) 대신 flat 포맷
- 진행률 표시 개선을 원하면 `pip install tqdm`

### 2-A. 로컬 모드로 보기

`file_tree_viewer.html`을 브라우저에서 열고, 만든 `.txt` 파일들을 드래그 앤 드롭하세요. 같은 prefix의 보조 파일을 함께 드롭하면 자동 매칭됩니다:

- `a_260527_2158.txt` ↔ `a.jpg` — 드라이브 썸네일
- `a_260527_2158.txt` ↔ `capacity.json` 내 `"a": { ... }` — 라벨·용량

### 2-B. 서버 모드로 보기

1. **캐시 폴더 준비** — 프로젝트 루트에 `data/` 폴더를 만들고 그 안에 모든 캐시 자산을 모으세요:

   ```
   data/
   ├── a_260527_2158.txt
   ├── a.jpg
   ├── b_260527_2200.txt
   ├── b.jpg
   └── capacity.json
   ```

2. **서버 실행**

   ```bash
   python server.py
   ```

   기본값: `data/` 디렉토리, 포트 `8765`, 모든 인터페이스에 바인드(LAN 접속 허용). 실행하면 로컬·LAN 접속 URL이 출력됩니다.

3. **브라우저로 접속**

   - 같은 PC: `http://127.0.0.1:8765/`
   - 다른 PC/폰: `http://<서버PC의 LAN IP>:8765/` (예: `http://192.168.0.10:8765/`)

   페이지가 열리면 자동으로 `data/` 안의 모든 캐시를 로드하고, 푸터에 "서버 모드" 뱃지가 표시됩니다. 서버 모드에서도 추가 파일을 드래그 앤 드롭해 일회성으로 더해 볼 수 있어요.

#### server.py 옵션

```bash
python server.py --data D:\drive-cache   # 캐시 폴더 지정
python server.py --port 8080             # 포트 변경
python server.py --host 127.0.0.1        # LAN 외부 노출 차단 (로컬만)
```

| 엔드포인트 | 동작 |
|------------|------|
| `GET /` | 뷰어 HTML |
| `GET /api/manifest` | 드라이브 목록 + 용량 정보 (JSON) |
| `GET /api/file/<name>` | 파일 목록 .txt 원본 |
| `GET /api/thumb/<name>` | 썸네일 이미지 |
| `GET /api/capacity` | capacity.json 원본 |

> 서버는 외부 인증 없이 캐시 디렉토리 내용을 노출합니다. 사적인 파일 목록이 들어있다면 신뢰 가능한 네트워크에서만 띄우거나 `--host 127.0.0.1`로 외부 접근을 막으세요.

## 주요 기능

- **멀티 드라이브 탭** — 여러 드라이브를 한 화면에서 전환하며 탐색
- **Dashboard** — 모든 드라이브의 썸네일·용량 게이지 한눈에
- **Merged 뷰** — 모든 드라이브를 동시에 검색·열람
- **빠른 필터링** — 이름/경로 부분 일치 검색, 결과 클릭 시 트리 위치로 점프
- **경로 복사** — 행 hover 시 나타나는 버튼으로 전체 경로 클립보드 복사
- **검색 결과 일괄 복사** — 필터 결과의 모든 경로를 한 번에 클립보드로
- **선택 합산** — 체크박스로 다중 선택 → 개수·총 용량 자동 집계
- **용량 게이지** — JSON의 capacity와 실제 합산 크기로 사용률 시각화
- **폴더만 보기 / 크기 표기 모드 토글**
- **서버 자동 로드** — 페이지 진입 시 캐시 매니페스트를 자동 fetch

## 출력 파일 포맷

기본은 **그룹 포맷** — 부모 경로를 한 번만 기록하고 파일을 들여쓰기로 표시해 텍스트가 작습니다.

```
# format: grouped, with_size
E:\[게임]\Pillowheads
	Pillowheads.exe	650752
	UnityPlayer.dll	22904264
E:\[게임]\다른게임
	game.exe	100000
```

- 탭으로 시작하지 않는 줄 → 폴더 절대 경로
- 탭으로 시작하는 줄 → 그 폴더 안의 파일 (`\t파일명\t크기`)
- 호환성이 필요하면 `--flat` 옵션으로 모든 줄을 풀패스로 저장 가능

## capacity JSON 포맷

드라이브별 라벨과 총 용량을 지정하면 뷰어에서 사용률 게이지와 친화적인 이름을 표시합니다.

```json
{
  "a": {
    "label": "External Drive A",
    "capacity": 5000000000000,
    "offset": 0
  },
  "b": {
    "label": "External Drive B",
    "capacity": 4000000000000,
    "offset": 0
  }
}
```

- 키(`a`, `b`)는 `.txt` 파일명의 prefix와 매칭 (예: `a_260527_2158.txt` → `a`)
- `capacity` — 드라이브 총 용량(bytes)
- `offset` — 파일 목록 합산과 실제 사용량의 차이 보정값 (선택)
- `label` — 뷰어 탭/대시보드에 표시할 친화적 이름 (선택)

배열 형태나 다른 키 이름(`bytes`, `total`, `size` 등)도 지원합니다. `capacity.example.json` 참고.

## 권장 워크플로우

1. 드라이브를 처음 연결할 때마다 `list_drive_files.py`로 목록 추출 → `<문자>_*.txt`로 저장
2. 각 드라이브 사진을 같은 prefix의 `.jpg`/`.png`로 저장 (선택)
3. 모든 드라이브의 라벨·용량 정보를 `capacity.json` 하나에 모으기
4. 모든 결과물을 `data/` 폴더로 모으고 `python server.py` 실행
5. 외장하드가 없는 곳에서도 브라우저로 검색

## 동작 환경

- Python 3.7+ (스크립트·서버 모두 표준 라이브러리만으로 동작 — `tqdm`은 진행률 표시용 옵션)
- 최신 브라우저 (Chrome / Edge / Firefox / Safari)
- 서버는 OS 무관 (Windows / macOS / Linux)
